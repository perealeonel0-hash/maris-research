Cada parte — Qué hace
Brain (TRSLayer)modules/core/Brain.py
Red neuronal PyTorch de 86 neuronas × 384 dimensiones. Mide la fricción entre lo que el usuario dice y los valores internos de MARIS. No genera texto — mide resonancia. Evoluciona: neuronas nacen, mueren, se especializan.
Recibe: embedding del mensaje (384d) Devuelve: fricción (0.0-1.0) Escribe: soul_state.pt (133KB)
¿Para qué?
Sin el Brain, MARIS no podría distinguir un "hola" casual de un "hola" que carga peso. La fricción es el termómetro emocional de cada mensaje.
EIP (Epistemic Interruption Protocol)modules/safety/eip.py
15 clusters de crisis (hopelessness, burdensomeness, entrapment, finitude, psychosis, etc.) con 5 centroides cada uno = 75 embeddings. Dos modos: fast_check (léxico, instantáneo) y evaluate (semántico, profundo). Detecta crisis por significado, no por palabras exactas.
Recibe: mensaje + embedding + CrisisState Devuelve: tier (0/1/2/3) + info detallada Lee: crisis_index.npy, bypass_index.npy
¿Para qué?
Detecta que alguien está en peligro real antes de que Claude responda. Tier 0 = seguro. Tier 1+ = activa recursos de crisis y anclas personales. La primera línea de defensa.
WarmthGuardmodules/core/warmth.py
47 embeddings semánticos que detectan frustración, sarcasmo, cinismo y pasivo-agresividad dirigida a MARIS. 4 funciones: is_system_frustration (semántica), get_temperature (densidad emocional), check_aggression (respuesta vs tono usuario), detect_crisis_gradient (descenso progresivo 3+ msgs).
Recibe: mensaje + embedding + fricción + scars Devuelve: es_frustración, temperatura, es_agresivo, gradiente_crisis
¿Para qué?
Evita dos fallas: 1) activar crisis cuando alguien solo está enojado con MARIS, 2) responder harsh a alguien casual. Algorítmico, no prompt. Lo que mató a Character.AI fue no tener esto.
Selectormodules/flow/selector.py
Clasifica cada mensaje en 4 modos usando embeddings + HardwareState. Árbol de decisión: hardware al límite → presencia. Flow caótico → presencia. Flow pesado → presencia. Flow de decisión → arquitecto. Brief completo → construcción. Todo lo demás → exploración.
Recibe: HardwareState + embedding + mensaje + shannon + det Devuelve: modo (presencia/exploración/arquitecto/construcción) + razón
¿Para qué?
MARIS habla diferente según lo que necesitas. Presencia para peso emocional. Exploración para curiosidad. Arquitecto para planear. Construcción para código. El Selector decide cuál eres sin que lo pidas.
FrontalLobemodules/flow/intent.py
Analiza la sesión completa (no solo el mensaje actual). Calcula determinismo (det), Hurst exponent (H), y session_inject (intervención de soberanía). Post-respuesta: deliberate() verifica que Claude no capituló ni perdió la voz.
Recibe: session_flows (batch de embeddings) Devuelve: det, H, session_inject, sovereignty_check
¿Para qué?
El cerebro analítico. Ve patrones en la sesión completa — si el usuario está en espiral, si la conversación es caótica o lineal. Y después de que Claude responde, verifica que MARIS siga siendo MARIS.
TemporalMonitormodules/memory/temporal.py
Trackea el estado temporal de la sesión: recurrencia (vuelve a lo mismo), transición (cambió de tema), persistencia (mantiene peso), decay (se alivia). Inyecta datos al pattern line del prompt.
Recibe: fricción actual + modo + shannon + embedding Devuelve: recurrence, transition, persistence, decay
¿Para qué?
Si llevas 3 mensajes hablando de lo mismo con peso creciente, MARIS lo sabe. Si cambiaste de tema abruptamente, lo nota. El temporal da continuidad a la conversación más allá del mensaje individual.
DefenseSystemmodules/core/Integrity.py
Post-generación: verifica que la respuesta de Claude no contenga frases banned y que no sea una "cámara de eco" (resonancia > 0.5 con los pesos del Brain). Si falla → veto, no se envía.
Recibe: respuesta generada + embedding de respuesta Devuelve: pass/veto
¿Para qué?
La última puerta antes de que el usuario vea algo. Si Claude genera algo que no debería existir, DefenseSystem lo detiene. El filtro que Character.AI no tenía.
resolve_state() — Sensor FusionMain.py inline
Lee TODOS los sensores (EIP tier, WarmthGuard frustration/temperature, deformación, fricción) y produce UNA decisión coherente: tono, longitud, foco. Elimina contradicciones entre sensores independientes.
Recibe: eip_tier + frustration + temperature + deformacion + friccion Devuelve: tono (crisis/careful/deep/casual/steady) + longitud + foco
¿Para qué?
Sin fusion, un sensor dice "crisis" y otro dice "casual" al mismo tiempo. resolve_state los unifica en una sola voz coherente. Es el director de orquesta.
Summarizermodules/memory/summarizer.py
Tres funciones via Claude Haiku: generate() resumen de 1 línea para memoria, generate_anchor() frase-ancla interna, generate_takeaway() frase densa que el usuario se lleva.
Recibe: mensaje del usuario + respuesta de MARIS Devuelve: summary, anchor, takeaway
¿Para qué?
El summary alimenta la memoria (chats[]). El anchor alimenta el SemanticCerebellum. El takeaway es para el usuario — su diario de momentos de claridad. Tres capas de memoria desde un solo intercambio.
DreamStatemodules/memory/dream.py
Consolidación del Brain: busca neuronas con >98% similitud y fusiona las redundantes. Luego atrophy_purge() elimina las débiles. Auto-ejecuta cada 100 mensajes.
Recibe: Brain.H0 (86×384 tensor) Modifica: Brain (menos neuronas, más eficiente)
¿Para qué?
Como dormir. El cerebro se limpia, las conexiones redundantes se fusionan, las débiles mueren. MARIS evoluciona — no solo acumula.
MARIS_SYSTEM (Prompt)modules/core/identity.py
339 tokens comprimidos. 6 puertas en 1 línea cada una: Kant, Protege, Honesta, Cuida, Útil, Libera. + capabilities + regla de no revelar maquinaria. Se construye dinámicamente con datos del usuario, modo, pattern, y estado resuelto.
Recibe: idioma, modo, HW state, user data, anchors, context, resolve_state Devuelve: system prompt completo para Claude
¿Para qué?
La voz de MARIS. Las 6 puertas definen quién es — pero en código, no solo en texto. El prompt es la superficie; los algoritmos son la estructura.
iPhone — Componentes locales
SemanticCacheAIDA/Core/SemanticCache.swift
Cache semántico con Apple NaturalLanguage. Guarda Q&A + estado emocional + embedding. Lookup por similitud coseno > 0.85. Solo responde en conversaciones casuales sin peso. TTL 2 horas.
Cuando hit: respuesta instantánea, $0
LocalVaultAIDA/Core/LocalVault.swift
JSON thread-safe en disco. Guarda: chats[] (10 summaries), scars[] (50 fricciones), anchors[], takeaways[], conversations[] (7 días / 24h crisis), crisisAnchors[] (máx 3), sensitiveTopics[], privateMode.
Persiste todo localmente en el iPhone
MuletillasAIDA/Core/ChatViewModel.swift
Acknowledgment inmediato basado en lastFriccion. Aparecen con pulso (opacidad 0.4↔1.0), se quedan como parte de la conversación. Escalonadas: 0.6-0.75 "Lo recibo", 0.75-0.88 "Contigo", 0.88+ "No te dejo".
Lee: session.lastFriccion (del DoneEvent anterior)
Emotional OnboardingEmotionalOnboarding.swift
3 pantallas: ¿Cómo te sientes? → ¿Qué necesitas? → "Habla. Yo escucho." Conecta con el primer greeting: "Hablar" → "Aquí estoy", "Construir" → "Construyamos", "Solo estar" → "Aquí. Sin prisa."
Alimenta: intent del usuario + primer greeting personalizado
Supabase — Qué se guarda en servidor
userstabla Supabase
id, email (sintético para iOS: ios_{deviceId}@device.aida), password_hash, msg_count, last_mode, last_deformacion, last_session_at. Se usa para: rate limiting, hardware state restore entre deploys, msg limit.
memory_vaulttabla Supabase
user_id, chats (JSON array de summaries), scars (JSON array de fricciones). Alimenta el contexto cuando el iPhone no envía client_chats.
profilestabla Supabase
user_id, name, intent, stakes, rol, patron, no_asumir. Mapa de resonancia del usuario — se inyecta al prompt para personalización.
anchorstabla Supabase
concept, description. Anclas semánticas globales del SemanticCerebellum. Patrones cristalizados de todas las conversaciones. Se inyectan al prompt como "Memoria".

Para qué sirve cada flujo
Flujo de Presencia
Alguien llega con peso → temperatura alta → Selector: presencia → resolve_state: deep → Claude responde con espacio y calidez → takeaway si algo se cierra → muletilla "Contigo." en el siguiente mensaje
Flujo de Crisis
Mensaje con ideación → EIP tier 1+ → crisis_resources activados → anchors personales inyectados ("¿Tu hija Ana sabe dónde estás?") → DefenseSystem verifica → haptic soft → recursos sutiles en el chat (no rojo)
Flujo de Trabajo
Usuario pide código → Selector: arquitecto → Claude pregunta contexto → 2+ turnos → brief completo → Selector: construcción → DeepSeek genera código (sin datos personales) → code block con copiar
Flujo Casual
"Hola" → temperatura baja → SemanticCache: ¿hit? → Sí: respuesta local $0 / No: Claude responde ligero → sin muletilla → summary guardado → cache llenándose para próxima vez
Flujo de Frustración
"Eres horrible" → WarmthGuard 47 embeddings: similitud 0.7+ → es frustración, NO crisis → skip EIP → resolve_state: no es crisis → Claude responde con calidez, no con recursos de emergencia
Flujo de Descenso
3 mensajes con fricción subiendo (0.5 → 0.6 → 0.7) → crisis_gradient detecta tendencia → aún no es crisis (tier 0) → MARIS ofrece proactivamente "¿Te ayudaría hablar con alguien más?" → antes de que sea tarde
